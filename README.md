# 📦 Products Table

React-приложение для управления каталогом товаров с авторизацией, таблицей продуктов, поиском, сортировкой и возможностью добавления новых товаров.

## ✨ Основные возможности

- 🔐 **Авторизация** — вход с запоминанием сессии
- 📋 **Таблица товаров** — адаптивное отображение с пагинацией
- 🔍 **Поиск** — мгновенный поиск по наименованию товаров
- ⬆️⬇️ **Сортировка** — сортировка по цене, рейтингу и наименованию
- ➕ **Добавление товаров** — модальная форма с валидацией
- 🔄 **Обновление данных** — ручное обновление списка товаров
- 📱 **Адаптивность** — корректная работа на всех устройствах
- 💾 **Локальное хранилище** — сохранение настроек сортировки

## 🚀 Технологии

- **React 19** + **TypeScript**
- **Vite 7** — сборка и dev-сервер
- **React Query (TanStack Query)** — работа с серверным состоянием
- **React Hook Form** + **Yup** — формы и валидация
- **Styled Components** — стилизация
- **React Router** — маршрутизация
- **Framer Motion** — анимации
- **React Hot Toast** — уведомления

## 🔌 Dummy JSON API

Приложение использует [Dummy JSON](https://dummyjson.com/) — бесплатный fake REST API для разработки и тестирования.

**Base URL:** `https://dummyjson.com`

### Авторизация

| Метод | Эндпоинт        | Описание                                                     |
| ----- | --------------- | ------------------------------------------------------------ |
| POST  | `/auth/login`   | Вход: `username`, `password`, `expiresInMins: 30` → токены в cookies |
| GET   | `/auth/me`      | Текущий пользователь (валидация сессии)                             |
| POST  | `/auth/refresh` | Обновление токена                                                   |

**Проверка авторизации:** ProtectedRoute валидирует сессию через `/auth/me`. Данные пользователя — в `localStorage`/`sessionStorage` (опция «Запомнить меня»). Время жизни токена — 30 минут.

**Тестовые данные:** можно использовать любые учётные записи из `/users` (например, `emilys` / `emilyspass`).

### Товары

| Метод | Эндпоинт                     | Описание                                      |
| ----- | ---------------------------- | --------------------------------------------- |
| GET   | `/products`                  | Список товаров с пагинацией (`limit`, `skip`) |
| GET   | `/products/search?q={query}` | Поиск товаров по наименованию                 |
| POST  | `/products/add`              | Добавление нового товара                      |

**Сортировка:** клиентская сортировка по полям `title`, `price`, `rating` (ASC/DESC). Выбранная сортировка сохраняется в `localStorage`.

**Поиск:** поиск по наименованию товара через API эндпоинт `/products/search`. Поддерживается сброс поиска и возврат к общему списку.

### Валидация форм

**Форма входа** (Yup):

- Логин — минимум 3 символа
- Пароль — минимум 6 символов

**Форма добавления товара** (Yup):

- Наименование — обязательное поле, минимум 3 символа
- Цена — обязательное поле, положительное число
- Вендор — обязательное поле, минимум 2 символа
- Артикул — обязательное поле, минимум 3 символа

## 🌐 Деплой на GitHub Pages

Приложение настроено на автоматический деплой на GitHub Pages при каждом пуше в ветку `master`.

### Настройка

1. В репозитории: **Settings** → **Pages**
2. В разделе **Build and deployment** выберите **GitHub Actions** как источник
3. После первого пуша в `master` сайт будет доступен по адресу:

```
  https://<username>.github.io/<repository-name>/
```

### Ручной запуск

Вкладка **Actions** → выберите workflow **Deploy to GitHub Pages** → **Run workflow**

## 🎯 Особенности реализации

### Архитектура

- **Feature-Sliced Design** — модульная архитектура с четким разделением слоев
- **Кастомные хуки** — переиспользуемая бизнес-логика (`useAuth`, `usePagination`, `useSort`, `useSearch`)
- **Композиция компонентов** — гибкие UI-компоненты с поддержкой различных состояний

### UX/UI

- **Адаптивный дизайн** — корректное отображение на мобильных устройствах и десктопах
- **Preloader** — индикация загрузки данных
- **Toast-уведомления** — информирование об успешных операциях и ошибках
- **Модальные окна** — блокировка скролла при открытии, закрытие по ESC и клику вне
- **Анимации** — плавные переходы с помощью Framer Motion

### Производительность

- **React Query** — кэширование запросов, автоматическое обновление при повторном фокусе
- **Оптимизация рендеров** — мемоизация колбэков
- **Lazy loading** — отложенная загрузка изображений

### Управление состоянием

- **Cookies** — токены авторизации (HttpOnly, 30 мин)
- **LocalStorage / SessionStorage** — данные пользователя для UI и настройки сортировки
- **URL state** — синхронизация состояния пагинации с URL (future feature)

## 📝 Примечания

- Приложение использует **Dummy JSON API**, поэтому добавление товаров симулируется (товар не сохраняется на сервере)
- Токен авторизации обновляется автоматически при истечении срока действия
- При выходе из системы все токены удаляются из хранилища
